# VARS — Claude Code Project Instructions

## Codebase context

Before making decisions that touch product behaviour, copy, payments, KYC, partnerships, or business logic (Phase 2 triggers, cancellation rules, notification strings), read **`docs/VARS_PROJECT_CONTEXT.md`**. It covers the things that aren't visible from code alone: the Product Spec location, current build state vs. in-flight work, all notification copy, Youverify and Monnify status, Phase 2 conditions, and the exact mechanics of the cancellation flag and vendor suspension system. If you're unsure whether something is built, check the README. If you're unsure what it's supposed to do, check the context doc.
